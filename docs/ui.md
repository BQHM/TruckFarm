# TruckFarm 页面与交互设计

版本：v0.1
日期：2026-05-12
状态：UI 草案

## 1. 设计目标

TruckFarm 的界面不应只是默认后台模板，而要体现“农场经营 + 数据管理”的产品感。

设计关键词：

- 清晰：经营指标、任务、预警一眼可见。
- 自然：色彩和视觉元素体现农业场景。
- 专业：表格、表单、权限和状态流转符合管理系统习惯。
- 可演示：核心流程可以在 3 到 5 分钟内完整演示。

## 2. 视觉方向

| 强调色 | `#D98E32` | 代表收获和土地 |

| 文本 | `#26332B` | 深绿色灰文字 |

### 2.2 字体和图标

- 中文优先使用系统可用字体，保证 Windows 和浏览器兼容。
- 标题使用更明确的字重和层级。
- 数字指标使用醒目的数字样式。
- 菜单图标应与业务相关：作物、地块、库存、订单、看板。

## 5. 全局布局

桌面端：

```text
+--------------------------------------------------+
| 顶栏：Logo / 农场名称 / 搜索 / 用户菜单          |
+---------------+----------------------------------+
| 侧边栏菜单    | 面包屑 / 页面标题 / 内容区        |
|               | 表格、表单、图表、详情            |
+---------------+----------------------------------+
```

要求：

- 侧边栏支持折叠。
- 顶栏显示当前用户和退出登录。
- 页面内容宽度自适应。
- 表格页保留搜索区、操作区、表格区和分页区。

移动端：

- 侧边栏变为抽屉菜单。
- 首页指标卡片改为单列或双列。
- 表格页可以横向滚动。
- 关键操作按钮固定在表格上方。

## 6. 核心页面设计

### 6.1 登录页

元素：

- 左侧：农场场景插画、渐变背景或几何地块纹理。
- 右侧：登录表单卡片。
- 标题：TruckFarm 智慧农场经营管理平台。
- 表单：账号、密码、验证码、登录按钮。
- 底部：演示账号提示。

交互：登录中显示 loading；登录失败展示明确错误；回车触发登录。

### 6.2 首页看板

首页必须讲清楚当前农场经营状态。

模块：

1. 顶部指标卡：种植面积、进行中计划、库存预警、本月收入。
2. 左侧主图：最近 6 个月销售趋势。
3. 右侧排行：作物销售贡献度。
4. 中部列表：待收获计划、库存预警。
5. 底部动态：最近采购、最近销售、最近库存流水。

图表：销售趋势用折线图或柱线混合图；作物贡献用横向柱状图；计划状态用环形图。

### 6.3 作物管理

搜索条件：作物名称、作物分类、状态。

表格列：作物名称、分类、生长周期、参考价格、单位、状态、更新时间、操作。

操作：新增作物、编辑、启用/禁用、查看关联种植计划。

### 6.4 地块管理

建议展示方式：表格 + 状态卡片。

表格列：地块编号、地块名称、面积、位置、土壤类型、当前状态、当前种植作物、操作。

状态颜色：空闲绿色，种植中蓝绿色，休耕橙色，停用灰色。

### 6.5 种植计划

搜索条件：作物、地块、状态、日期范围。

操作按钮根据状态显示：

| 状态 | 可操作 |
| --- | --- |
| 待开始 | 编辑、开始种植、取消 |
| 种植中 | 完成收获、取消 |
| 已收获 | 查看收获记录 |
| 已取消 | 查看 |

完成收获弹窗字段：实际收获日期、产量、损耗、单位、备注。

### 6.6 采购订单

创建流程：

1. 选择供应商。
2. 选择负责人和采购日期。
3. 添加采购明细。
4. 自动计算总金额。
5. 保存为待入库。
6. 点击入库生成库存流水。

状态操作：草稿可编辑和提交；待入库可入库或取消；已入库和已取消只能查看。

### 6.7 库存管理

表格列：作物、当前库存、单位、预警阈值、预警状态、最近变动时间、操作。

操作：查看流水、盘点调整、修改预警阈值。

### 6.8 销售订单

创建流程与采购类似，但出库时需要校验库存。

关键交互：

- 明细选择作物时显示当前库存。
- 输入销售数量超过库存时前端提示，后端仍必须校验。
- 出库成功后订单状态变为已完成。

### 6.9 系统管理

- 用户管理：用户状态、角色分配、重置密码。
- 角色管理：角色信息和菜单授权。
- 菜单管理：树形表格。
- 操作日志：按模块、操作人、时间查询。

## 7. 表格页通用规范

每个表格页包含搜索表单、工具栏、数据表格、分页、新增或编辑弹窗。

要求：

- 搜索条件支持重置。
- 表格加载中显示 loading。
- 空数据展示业务友好的空状态。
- 删除或取消操作必须二次确认。
- 状态字段使用 Tag 展示。
- 金额和数量右对齐。
- 日期格式统一为 `YYYY-MM-DD` 或 `YYYY-MM-DD HH:mm:ss`。

## 8. 表单规范

- 必填项必须标记。
- 前端做基础校验，后端做最终校验。
- 金额、数量、面积使用数字输入组件。
- 日期使用日期选择器。
- 提交中禁用按钮，避免重复提交。
- 成功后提示并刷新列表。

## 9. 权限交互

- 无菜单权限时不显示对应菜单。
- 无按钮权限时不显示对应按钮。
- 直接访问无权限路由时展示 403 页面。
- Token 失效时跳转登录页。

## 10. 演示路径

1. 使用管理员账号登录。
2. 查看首页经营看板。
3. 新增一个作物和地块。
4. 创建种植计划并开始种植。
5. 创建采购单并入库，查看库存增加。
6. 创建销售订单并出库，查看库存扣减。
7. 回到首页查看销售趋势和库存预警变化。
8. 展示角色菜单权限和操作日志。
