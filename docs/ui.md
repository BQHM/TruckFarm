# TruckFarm 页面与交互设计

版本：v0.2
日期：2026-05-13
状态：UI 草案

## 1. 设计目标

TruckFarm 的界面不应只是默认后台模板，而要体现“农场经营 + 数字沙盘 + 经营复盘”的产品感。

设计关键词：

- 清晰：经营指标、任务、预警一眼可见。
- 自然：色彩和视觉元素体现农业场景。
- 专业：表格、表单、权限和状态流转符合管理系统习惯。
- 可演示：核心流程可以在 3 到 5 分钟内完整演示。
- 沙盘感：用户能看到日期推进、天气变化、地块状态、事件触发、资金变化和经营结果。

## 5. 全局布局

桌面端：

```text
+--------------------------------------------------------------+
| 顶栏：Logo / 当前日期天气 / 农场名称 / 用户菜单              |
+----------------+---------------------------------------------+
| 侧边栏菜单     | 面包屑 / 页面标题 / 内容区                   |
|                | 沙盘卡片、表格、图表、时间线、详情面板       |
+----------------+---------------------------------------------+
```

要求：

- 侧边栏支持折叠。
- 顶栏显示当前用户和退出登录。
- 顶栏显式显示当前模拟日期、天气图标和季节。
- 页面内容宽度自适应。
- 表格页保留搜索区、操作区、表格区和分页区。

移动端：

- 侧边栏变为抽屉菜单。
- 首页指标卡片改为单列或双列。
- 表格页可以横向滚动。
- 关键操作按钮固定在页面上方。

## 6. 核心页面设计

### 6.1 登录页

元素：

- 左侧：农场场景插画、渐变背景或几何地块纹理。
- 右侧：登录表单卡片。
- 标题：TruckFarm 农场经营数字沙盘。
- 表单：账号、密码、验证码、登录按钮。
- 底部：演示账号提示。

交互：登录中显示 loading；登录失败展示明确错误；回车触发登录。

### 6.2 首页沙盘

首页必须讲清楚当前农场经营状态，并让用户感知这是一个会随时间变化的沙盘。

模块建议：

1. 顶部状态条：当前日期、季节、天气、经营轮次。
2. 第一屏指标卡：种植面积、进行中计划、库存预警、当前余额、本月收入、本月成本。
3. 左侧主区域：地块沙盘缩略图或关键地块卡片。
4. 右侧事件栏：天气风险、病虫害事件、市场波动、今日待办。
5. 中部图表：销售趋势 + 资金趋势双图。
6. 底部列表：待收获计划、库存预警、最近动作结果、最近资金流水。

首页要有叙事感：让用户一眼看出“今天发生了什么、赚了多少钱、哪里有风险、下一步该做什么”。

### 6.3 地块沙盘

展示方式建议：2D 地块卡片网格或简化平面图。

每个地块展示：

- 地块名称和编号。
- 当前种植作物。
- 生长进度。
- 健康度。
- 湿度 / 肥力 / 病虫害风险。
- 当前状态颜色。

点击地块进入详情侧栏：展示最近 7 天天气影响、操作记录和预估产量变化。

### 6.4 时间线与事件流

页面目标：让用户看到时间推进带来的变化。

模块：

- 日期推进控制区：推进一天、推进三天、自动推进开关。
- 时间线主体：天气、事件、操作结果按时间排序。
- 过滤器：事件类型、风险等级、地块、作物。
- 详情抽屉：展示事件影响对象、影响结果和关联 outcome。

### 6.5 经营动作页

需要集中提供沙盘相关动作入口：

- 采购。
- 播种。
- 浇水。
- 施肥。
- 除虫。
- 收获。
- 销售。

交互要求：

- 左侧表单输入动作参数。
- 右侧显示当前上下文提示，如天气、库存、地块健康度、客户或供应商状态。
- 提交后跳转动作结果页，说明“为什么得到这个结果”。

### 6.6 动作结果页

用户执行采购、播种、养护、收获或销售后，需要展示“为什么得到这个结果”。

结果页内容：

- 操作摘要：动作类型、对象、数量、模拟日期。
- 环境上下文：天气、季节、地块状态、供应商或客户参数。
- 基础规则：本地规则计算出的优品率、损耗率、延迟、价格等区间。
- AI 修正：AI 给出的修正因子、风险等级和解释；AI fallback 时展示本地规则兜底。
- 最终结果：优品数量、普通数量、损耗数量、延迟天数、成交价格等。
- 资金影响：收入 / 支出金额、余额变化。
- 事件记录：本次操作触发的农场事件。
- 复现信息：random seed 和 outcomeId，默认折叠。

### 6.7 作物管理

搜索条件：作物名称、作物分类、状态。

表格列：作物名称、分类、生长周期、参考价格、单位、状态、更新时间、操作。

操作：新增作物、编辑、启用/禁用、查看生长规则、查看关联种植计划。

### 6.8 地块管理

建议展示方式：表格 + 状态卡片。

表格列：地块编号、地块名称、面积、位置、土壤类型、当前状态、当前种植作物、操作。

状态颜色：空闲绿色，种植中蓝绿色，休耕橙色，停用灰色。

### 6.9 种植计划

搜索条件：作物、地块、状态、日期范围。

操作按钮根据状态显示：

| 状态 | 可操作 |
| --- | --- |
| 待开始 | 编辑、开始种植、取消 |
| 种植中 | 完成收获、取消 |
| 已收获 | 查看收获记录 |
| 已取消 | 查看 |

完成收获弹窗字段：实际收获日期、产量、损耗、单位、备注。

### 6.10 采购订单

创建流程：

1. 选择供应商。
2. 选择负责人和采购日期。
3. 添加采购明细。
4. 自动计算总金额。
5. 保存为待入库。
6. 点击入库生成库存流水和资金支出流水。

### 6.11 销售订单

创建流程与采购类似，但出库时需要校验库存。

关键交互：

- 明细选择作物时显示当前库存。
- 输入销售数量超过库存时前端提示，后端仍必须校验。
- 出库成功后订单状态变为已完成，并产生资金收入。

### 6.12 资金页

资金页由两个视图构成：

- 资金账户：当前余额、本月收入、本月成本、净现金变化。
- 资金流水：时间、业务类型、摘要、收入 / 支出金额、变动后余额。

图表建议：

- 30 天资金趋势折线图。
- 收入 / 成本构成柱状图。

### 6.13 经营复盘页

经营复盘页用于展示一轮模拟的最终效果。

模块：

- 收入、成本、净利润。
- 资金流入、流出和现金余额变化。
- 作物产量和优品率排行。
- 采购损耗、库存损耗和销售收益。
- 关键风险事件时间线。
- 用户决策影响：及时处理、未处理、造成收益变化的动作。
- AI 经营总结：只展示建议，不覆盖事实数据。

## 7. 表格页通用规范

每个表格页包含搜索表单、工具栏、数据表格、分页、新增或编辑弹窗。

要求：

- 搜索条件支持重置。
- 表格加载中显示 loading。
- 空数据展示业务友好的空状态。
- 删除或取消操作必须二次确认。
- 状态字段使用 Tag 展示。
- 金额和数量右对齐。
- 日期格式统一为 `YYYY-MM-DD` 或 `YYYY-MM-DD HH:mm:ss`。

## 8. 表单规范

- 必填项必须标记。
- 前端做基础校验，后端做最终校验。
- 金额、数量、面积使用数字输入组件。
- 日期使用日期选择器。
- 提交中禁用按钮，避免重复提交。
- 成功后提示并刷新列表。

## 9. 权限交互

- 无菜单权限时不显示对应菜单。
- 无按钮权限时不显示对应按钮。
- 直接访问无权限路由时展示 403 页面。
- Token 失效时跳转登录页。

## 10. 演示路径

1. 使用管理员账号登录。
2. 查看首页沙盘，确认日期、天气、余额和风险。
3. 新增一个作物和地块，查看生长规则。
4. 创建采购动作或采购单并入库，查看库存和资金变化。
5. 创建种植计划并开始种植。
6. 推进一天，观察天气、地块和事件变化。
7. 收获入库或创建销售动作，查看动作结果页。
8. 打开经营复盘页，展示收入、成本、风险与决策影响。
