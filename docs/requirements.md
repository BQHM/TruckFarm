# TruckFarm 需求说明

版本：v0.1
日期：2026-05-12
状态：迭代升级需求草案

## 1. 项目背景

TruckFarm 原始版本是基于 Servlet、MiniUI 和 MySQL 的毕业设计项目，功能集中在登录、用户、部门、农作物管理等基础后台能力。本次不是纯粹新开发，而是在旧项目基础上进行迭代升级：边检查旧代码不规范之处，边学习 Java 25、Spring Boot 4、PostgreSQL、Flyway、Spring Security、Vue 3 等新技术，边将旧功能重构为现代前后端分离架构，并补足完整业务闭环和工程化能力。MyBatis / MyBatis Plus、OpenAPI 等生态依赖需要在 P1 阶段先做 Spring Boot 4 兼容性验证。

## 2. 产品定位

TruckFarm 面向中小型农场、农业合作社或农产品经营团队，帮助用户管理作物档案、地块、种植计划、采购库存、销售订单和经营看板。

目标：

- 作物资料可维护。
- 地块状态可追踪。
- 种植计划可流转。
- 采购入库和销售出库能影响库存。
- 首页看板能展示经营状态。
- 权限、日志、接口文档、Docker 部署具备简历项目完整度。

## 3. 用户角色

| 角色 | 关注点 |
| --- | --- |
| 农场负责人 | 经营概览、销售收入、库存预警、种植进度 |
| 种植管理员 | 作物档案、地块状态、种植计划、收获记录 |
| 采购管理员 | 供应商、采购单、采购入库、采购成本 |
| 仓库管理员 | 当前库存、库存流水、库存预警、盘点调整 |
| 销售管理员 | 客户、销售订单、销售出库、收入统计 |
| 系统管理员 | 用户、角色、菜单、权限、字典、日志 |

## 4. MVP 范围

第一版优先打通以下闭环：

```text
登录 -> 动态菜单 -> 作物档案 -> 地块管理 -> 种植计划 -> 采购入库 -> 库存管理 -> 销售出库 -> 首页看板
```

MVP 模块：

- 认证授权：登录、JWT、刷新 Token、退出登录、密码加密。
- 用户权限：用户、角色、菜单、角色菜单授权、动态路由。
- 组织员工：部门、员工资料、员工状态。
- 作物管理：作物分类、作物档案、生长周期、参考价格。
- 地块管理：地块档案、面积、位置、状态。
- 种植计划：计划创建、开始种植、完成收获、取消计划。
- 采购管理：供应商、采购单、采购明细、采购入库。
- 库存管理：当前库存、出入库流水、库存预警、盘点调整。
- 销售管理：客户、销售订单、销售明细、销售出库。
- 数据看板：经营概览、销售趋势、作物排行、待办和预警。
- 系统日志：登录日志、操作日志。

二期模块：文件附件、AI 辅助、通知提醒、多农场能力。

## 5. 核心功能

### 5.1 认证授权

- 用户使用账号密码登录。
- 密码使用 BCrypt 等安全算法加密存储。
- 登录成功返回 Access Token 和 Refresh Token。
- 退出登录后 Token 加入 Redis 黑名单。
- 后端返回当前用户、角色、权限和菜单。
- 未登录返回 401，无权限返回 403。

### 5.2 用户角色菜单

- 用户可绑定多个角色。
- 角色可绑定菜单和按钮权限。
- 前端根据后端菜单生成侧边栏和路由。
- 用户、角色、菜单变更需要记录操作日志。

### 5.3 作物与地块

- 作物支持分类、名称、生长周期、参考价格、单位、状态。
- 地块支持编号、名称、面积、位置、土壤类型、状态。
- 地块状态包括空闲、种植中、休耕、停用。
- 被业务引用的作物和地块禁止物理删除。

### 5.4 种植计划

- 计划必须绑定作物、地块和负责人。
- 状态流转：待开始 -> 种植中 -> 已收获，可取消。
- 预计收获日期不得早于计划开始日期。
- 同一地块同一时间段不能存在冲突的进行中计划。
- 完成收获时生成收获记录，保留产量、损耗和收获日期。

### 5.5 采购库存

- 采购单绑定供应商和采购负责人。
- 采购单包含多个明细，入库后增加库存并生成库存流水。
- 库存数量不得为负。
- 支持库存预警阈值，低于阈值展示在首页看板。

### 5.6 销售订单

- 销售单绑定客户和销售负责人。
- 销售单包含多个明细，出库时扣减库存并生成库存流水。
- 库存不足时禁止出库。
- 已取消订单不参与销售收入统计。

### 5.7 首页看板

首页展示：当前种植面积、进行中的种植计划数量、待收获计划数量、库存预警数量、本月采购成本、本月销售收入、最近 6 个月销售趋势、作物销售贡献度排行、待处理事项列表。

## 6. 非功能需求

| 类别 | 要求 |
| --- | --- |
| 安全 | JWT 鉴权、密码加密、权限校验、敏感日志脱敏 |
| 性能 | 常规列表小于 500ms，首页看板小于 1s |
| 可维护性 | 前后端分离、模块化、统一异常、统一响应 |
| 技术验证 | P1 先验证 Java 25 + Spring Boot 4 + PostgreSQL + Flyway + Redis + MyBatis / MyBatis Plus + OpenAPI 兼容性 |
| 可部署性 | Docker Compose 一键启动 PostgreSQL、Redis、后端、前端 |
| 可测试性 | 核心 Service 有单元测试，关键接口有集成测试 |
| 可展示性 | README 有截图、演示账号、启动步骤和项目亮点 |

## 7. 不做范围

- 不做真实支付、物流和电商交易。
- 不接入真实农业传感器和物联网设备。
- 不做复杂财务总账和会计凭证。
- MVP 不做多租户 SaaS。
- AI 功能不作为核心业务强依赖。

## 8. 验收标准

- 可以登录系统并看到动态菜单。
- 可以完成作物、地块、种植、采购、库存、销售的完整演示流程。
- 首页看板能基于演示数据展示关键指标和图表。
- OpenAPI 文档可访问。
- Docker Compose 可以启动基础环境。
- README 有启动步骤、演示账号和截图。
